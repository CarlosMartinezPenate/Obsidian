# PLAN DE GOBERNANZA CONSULTORÍA  BASADA EN DATOS
# Fase 0 - Arquitectura mínima de control operativo - Parrilla Project

## 1 - PRINCIPIOS FUNDAMENTALES
1. Separación clara entre el liderazgo técnico y el liderazgo comercial.
2. Reunión multipartita antes de iniciar.  
3. Alcance cerrado antes de iniciar.
4. Cambios de alcance solo por escrito.
5. Las decisiones técnicas pertenecen al liderazgo técnico. 
6. El cliente no redefine la metodología.
## 2 - ESTRUCTURA DE ROLES
###  Lead Técnico
Autoridad sobre:
- Modelo de datos.
- Definición de KPI.
- Métodos de limpieza.
- Validez técnica.
- Documento de hallazgos.
Responsabilidad:
- Entregar estructura de proyecto definida.
- Documentar y definir limitaciones.
- Ejecutar estructura de proyecto
- Entrega de resultados
###  Lead Comercial
Autoridad sobre:
- Comunicación con cliente.
- Agenda.
- Cobro.
- Seguimiento.
Responsabilidad:
- Conseguir información completa.
- Proteger el marco aprobado.
- Comunicar resultados al cliente.

## 4 -  MECANISMO DE CONTROL DE ALCANCE

Si el cliente pide algo adicional:

1. Se documenta.
2. Se evalúa impacto técnico.
3. Se estiman horas.
4. Se cotiza adicional.

Nunca:
“Démoselo por esta vez.”

## 6 - MECANISMO DE RESOLUCIÓN DE CONFLICTOS

Si hay desacuerdo:

Paso 1: Se discute técnicamente.

Paso 2: Se documenta postura.

Paso 3: Si es técnico → Carlos decide

Paso 4: Si es comercial → Jose decide

**Sin descalificación personal. Si aparece descalificación personal hay pausa del proyecto.**


## 7 - REGLA DE SALIDA

Si la dinámica se vuelve improductiva, cualquiera puede retirarse entregando lo trabajado hasta ese punto.

## 8 - PROTECCIÓN DE REPUTACIÓN
Nunca:

- Se habla mal del socio frente al cliente.
- Se contradice metodología frente al cliente.
- Se improvisa

## 9 - CRITERIO DE ÉXITO

El proyecto se considera exitoso si:
- Se puede responder: ¿Cuánto factura realmente el negocio?
- Se puede estimar Food Cost preliminar.
- Se detectan 3–5 brechas críticas.
- El cliente entiende/acepta la siguiente fase.
